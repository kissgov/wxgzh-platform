### 2.5 已知坑 (S3 经验,给 S4 警示)

- `tsconfig.tsbuildinfo` 不要 stage (V1 习惯,build artifact)
- `pnpm test swagger-sync` 仍需 `--forceExit` (BullModule 真实 Redis 问题,S1 既有)
- pnpm-lock.yaml 每次装包都会更新,commit 时一并带上
- `app.setGlobalPrefix('api/v1')` 后,/metrics 必须在 `setGlobalPrefix` 之前注册才能挂根路径
- OTel SDK 启动位置:当前在 main.ts 顶部,所有 import 之后调用。理论应在最顶部 module-level 调用,但 V1 的 `zod-to-swagger` 必须在 OTel 之前(否则 V1 的 zod 扩展丢失)。保持现状。
- /metrics 暴露 contentType 需用 `metricsRegistry.contentType` (text/plain; version=0.0.4)
- pino redact 走 `*.field` 通配,会匹配嵌套对象里所有同名字段
